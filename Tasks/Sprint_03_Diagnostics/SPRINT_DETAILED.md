# Детальний опис спринта

## Спринт
- # Sprint 03 — Diagnostics (2 недели)

## Що робимо
- Реализовать диагностическую интеллектуальную часть: epistemic-анализ, локализация проблем, problem factory, приоритизация вопросов.
- Виконуємо задачі спринта:
- `S3-T1`: Epistemic analysis (evidence/trust/uncertainty)
- `S3-T2`: Problem localization + Problem factory
- `S3-T3`: Question prioritization + Dialogue request artifacts

## Як виконуємо
- Через задачі у теках S*-T*/ (для кожної: BOARD.md + DETAILED.md).
- Порядок виконання визначається пріоритизацією:
- Must: S3-T1, S3-T2
- Should: S3-T3
- Усі артефакти ведемо файлово в межах проекту (v1 без БД).

## Очікуваний результат
- Система выдает внятную диагностическую картину и следующий лучший вопрос, опираясь на доказательства.

## Точки контролю
- Закриті всі задачі рівня Must для спринта.
- Для кожної задачі є заповнені BOARD.md і DETAILED.md.
- Артефакти спринта готові до передачі у наступний спринт без втрати контексту.
- Збережено обмеження v1: локальна робота через IDE, без підключення БД.
