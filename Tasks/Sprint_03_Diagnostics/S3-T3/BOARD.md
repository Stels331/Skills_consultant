# S3-T3 — Question prioritization + Dialogue request artifacts (общее описание)

**Для борди:** короткий опис задачі для швидкого ознайомлення.


**Приоритет:** Should


## Цель
Выбирать следующий лучший вопрос, который максимизирует снижение критической неопределенности.

## Что делаем
- Скоринг unknowns (impact/resolvability/cascade/urgency).
- Генерируем question queue и rationale.
- Поддерживаем правило: один вопрос -> один узел неопределенности.

## Критерий успеха
Система задает минимально достаточные и нерепетитивные вопросы.
