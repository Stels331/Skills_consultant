# S2-T1 — Intake + Parsing pipeline (общее описание)

**Для борди:** короткий опис задачі для швидкого ознайомлення.


**Приоритет:** Must


## Цель
Превратить сырой вход (текст/документы/ответы) в нормализованное представление для extraction.

## Что делаем
- Intake входов Analysis и Dialogue режимов.
- Нормализацию текста, сегментацию, извлечение метаданных.
- Запись `source_manifest.json`, `case_brief.json`, `parsed/*.json`.

## Критерий успеха
Все входы преобразуются в единый формат, пригодный для автоматической extraction.
