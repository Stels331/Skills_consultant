# Детальний опис спринта

## Спринт
- # Sprint 02 — Core Pipeline (2 недели)

## Що робимо
- Собрать ядро первичной обработки: intake, parsing, extraction, typization, case modeling.
- Виконуємо задачі спринта:
- `S2-T1`: Intake + Parsing pipeline
- `S2-T2`: Extraction + Typization + Glossary binding
- `S2-T3`: Canonical case model + index + базовое versioning

## Як виконуємо
- Через задачі у теках S*-T*/ (для кожної: BOARD.md + DETAILED.md).
- Порядок виконання визначається пріоритизацією:
- Must: S2-T1, S2-T2
- Should: S2-T3
- Усі артефакти ведемо файлово в межах проекту (v1 без БД).

## Очікуваний результат
- Из сырого входа формируется структурированный case model и индекс, пригодные для дальнейшей диагностики.

## Точки контролю
- Закриті всі задачі рівня Must для спринта.
- Для кожної задачі є заповнені BOARD.md і DETAILED.md.
- Артефакти спринта готові до передачі у наступний спринт без втрати контексту.
- Збережено обмеження v1: локальна робота через IDE, без підключення БД.
