# S6-T1 — Unit/integration/scenario test suite (глубокое описание)

## Мета
- Сформульована у задачі та узгоджена з цілями спринта.

## Що робимо
- Виконуємо обсяг робіт, описаний у цій задачі (див. робочі деталі нижче).

## Як виконання
- Реалізація по кроках із фіксацією артефактів у файловій структурі проекту.

## Очікуваний результат
- Отримано завершений артефакт задачі, готовий для інтеграції в потік спринта.

## Як тестувати виконання
- Запустити перевірки/скрипти модуля задачі локально в IDE.
- Перевірити, що всі вихідні файли задачі створені/оновлені коректно.
- Перевірити, що зміни не ламають попередній workflow спринта.

## Контрольні точки поза тестуванням
- Актуальність документації та читабельність артефактів для handoff.
- Відповідність архітектурним правилам (problem-first, evidence-first, no-DB v1).
- Відмітка прогресу задачі та синхронізація зі статусом спринта.

## Робочі деталі (початковий опис задачі)

**Приоритет:** Must


## Scope
- Unit: workspace, schema validation, router transitions, context budget, incremental update.
- Integration: intake -> model -> epistemic -> problem factory.
- Scenario: полный кейс с противоречиями и последующим решением.

## Технические шаги
1. Подготовить фикстуры кейсов в `tests/scenarios/fixtures`.
2. Написать smoke и regression тесты для каждого спринтового результата.
3. Добавить тесты на отсутствие DB-зависимостей.
4. Сформировать отчет покрытия и список рисков непокрытого функционала.

## Нефункциональные требования
- Тесты запускаются из IDE/CLI локально.
- Предсказуемое время прогона тестового набора.
- Независимость от внешней инфраструктуры.

## Acceptance
- Зеленый прогон для основного regression-набора.
- Критические сценарии (противоречия, unknowns, selection gates) покрыты.
