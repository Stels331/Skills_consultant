# S6-T2 — IDE-first developer experience (без БД) (глубокое описание)

## Мета
- Сформульована у задачі та узгоджена з цілями спринта.

## Що робимо
- Виконуємо обсяг робіт, описаний у цій задачі (див. робочі деталі нижче).

## Як виконання
- Реалізація по кроках із фіксацією артефактів у файловій структурі проекту.

## Очікуваний результат
- Отримано завершений артефакт задачі, готовий для інтеграції в потік спринта.

## Як тестувати виконання
- Запустити перевірки/скрипти модуля задачі локально в IDE.
- Перевірити, що всі вихідні файли задачі створені/оновлені коректно.
- Перевірити, що зміни не ламають попередній workflow спринта.

## Контрольні точки поза тестуванням
- Актуальність документації та читабельність артефактів для handoff.
- Відповідність архітектурним правилам (problem-first, evidence-first, no-DB v1).
- Відмітка прогресу задачі та синхронізація зі статусом спринта.

## Робочі деталі (початковий опис задачі)

**Приоритет:** Should


## Scope
- Обновить `README.md` и `docs/` с workflow "от клона до первого кейса".
- Подготовить make/task-команды для основных операций.
- Добавить шаблон `.env.example` только для опциональных интеграций.

## Технические шаги
1. Описать quickstart: setup, first analyze, first dialogue turn, reports.
2. Добавить единый entrypoint-скрипт для локального запуска.
3. Подготовить troubleshooting раздел по частым ошибкам.
4. Внедрить preflight-check (папки, права, схемы, зависимости).

## Нефункциональные требования
- Нулевое требование поднятия БД/очередей/кэшей.
- Все шаги валидны для запуска в IDE терминале.
- Минимальный когнитивный overhead для нового участника команды.

## Acceptance
- Onboarding в чистом окружении занимает <=30 минут.
- Все базовые команды выполняются локально без внешних сервисов.
