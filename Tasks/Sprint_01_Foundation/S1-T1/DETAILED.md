# S1-T1 — Файловый workspace и lifecycle кейса (глубокое описание)

**Статус:** DONE

## Мета
- Сформульована у задачі та узгоджена з цілями спринта.

## Що робимо
- Виконуємо обсяг робіт, описаний у цій задачі (див. робочі деталі нижче).

## Як виконання
- Реалізація по кроках із фіксацією артефактів у файловій структурі проекту.

## Очікуваний результат
- Отримано завершений артефакт задачі, готовий для інтеграції в потік спринта.

## Як тестувати виконання
- Запустити перевірки/скрипти модуля задачі локально в IDE.
- Перевірити, що всі вихідні файли задачі створені/оновлені коректно.
- Перевірити, що зміни не ламають попередній workflow спринта.

## Контрольні точки поза тестуванням
- Актуальність документації та читабельність артефактів для handoff.
- Відповідність архітектурним правилам (problem-first, evidence-first, no-DB v1).
- Відмітка прогресу задачі та синхронізація зі статусом спринта.

## Робочі деталі (початковий опис задачі)

**Приоритет:** Must


## Scope
- Реализовать модуль `workspace_manager`.
- Стандартизировать путь хранения: `cases/case_YYYYMMDD_NNN/`.
- Добавить состояния и переходы lifecycle в `workspace_metadata.json`.

## Артефакты
- `cases/<id>/raw/`
- `cases/<id>/parsed/`
- `cases/<id>/extracted/`
- `cases/<id>/model/`
- `cases/<id>/analysis/`
- `cases/<id>/dialogue/`
- `cases/<id>/evidence/`
- `cases/<id>/quality/`
- `cases/<id>/reports/`
- `cases/<id>/state/`
- `cases/<id>/versions/`

## Технические шаги
1. Добавить API/CLI функции: `create_workspace`, `load_workspace`, `set_workspace_state`, `archive_workspace`.
2. Валидировать обязательные файлы при загрузке workspace.
3. Реализовать snapshot-крючок для ключевых точек (решение/структурное изменение).
4. Логировать события lifecycle в `version_changelog.json`.

## Нефункциональные требования
- Без БД: только filesystem operations.
- Идемпотентность создания (повторный запуск не ломает структуру).
- Портируемость путей для локальной IDE-разработки.

## Acceptance
- Workspace создается <2 сек.
- Все обязательные каталоги и файлы присутствуют.
- Переходы lifecycle отражаются в changelog.
