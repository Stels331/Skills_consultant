## Контрольні точки поза тестуванням

- Відповідність архітектурним правилам (problem-first, evidence-first, no-DB v1).
- Відмітка прогресу задачі та синхронізація зі статусом спринта.

## Робочі деталі (початковий опис задачі)

**Приоритет:** Must


## Scope
- Поддержка входов: answer/question/document.
- Context selection по фазе роутера и связям из `model_index.json`.
- Лимиты: warning 8k, hard limit 10k tokens.

## Технические шаги
1. Реализовать менеджер диалога и event-log.
2. Построить модуль selection fragments by IDs.
3. Добавить счетчики загрузки контекста по turn.
4. Добавить fallback: compression + minimal reload при превышении лимита.

## Нефункциональные требования
- Быстрый отклик turn-level.
- Детальная наблюдаемость причин роста контекста.
- Полная локальная работа без Redis/DB session store.

## Acceptance
- >90% turn-ов на сценариях проходят ниже 10k токенов.
- Нет повторной полной загрузки кейса при обычном диалоге.
