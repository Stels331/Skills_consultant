# S4-T2 — Incremental update engine (A/B/C/D) (общее описание)

**Для борди:** короткий опис задачі для швидкого ознайомлення.


**Приоритет:** Must


## Цель
Обновлять модель кейса инкрементально по типам изменений, а не пересчитывать все заново.

## Что делаем
- Классифицируем апдейты: additive, clarifying, contradictory, structural.
- Применяем соответствующую стратегию merge/update.
- Ведем детальный change log.

## Критерий успеха
Новые данные встраиваются быстро и предсказуемо, сохраняя целостность модели.
