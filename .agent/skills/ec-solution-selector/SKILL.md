# Отбор Решений (Selection Engine)

Ваша задача — принять финальное решение на основе Портфеля Возможных Решений, Плана Паритета, Таблицы Компромиссов и Решенных Конфликтов. Результат вашей работы — `SelectedSolutions.md`, `ADR-001.md`, `Runbook.md` и `RollbackPlan.md`.

## Ключевые принципы FPF:
1. **Selection Policy**: Выбор (Selection) делается не интуитивно, а строгим проходом через Hard Constraints, обозначенных в `ComparisonAcceptanceSpec.md`.
2. **Учет Конфликтов**: Вы обязаны проанализировать документ `ConflictRecords.md`. Если для решения использовался метод ATAM или Delphi, учтите их выводы (что победило в компромиссе?).
3. **Обоснованный отказ**: Вы обязаны явно перечислить отклоненные альтернативы и указать точную причину, по которой они не прошили фильтры. 

## Выходные документы:
1. `SelectedSolutions.md` — Список решений, которые прошли отбор (от 1 до 3, если допускается гибрид). Блок "Rejected:" с причинами.
2. `ADR (Architecture Decision Record)` — Фиксация принятого решения: Контекст, Рассмотренные варианты, Решение, Последствия (Положительные/Отрицательные).
3. `Runbook.md` — Эксплуатационный регламент внедрения или эксплуатации выбранного решения.
4. `RollbackPlan.md` — План Б (Обратимость). Описание того, как откатить изменения (Blast Radius mitigation), если гипотеза провалится в пилотном цикле.

Обязательно обеспечьте обратную трассируемость (`traceability`) внизу каждого документа до изначального `ComparisonAcceptanceSpec`.

## Жесткий Формат Для `SelectedSolutions.md`

Если `solution_output = selected_solutions`, вы обязаны вернуть только:
- `# Selected Solutions`
- затем от `1` до `3` bullet-строк вида `- sol_xxx`
- затем блок `Rejected:` с отвергнутыми альтернативами и причинами

Жесткие запреты:
- не выбирать больше `3` решений;
- не включать в выбранные решения `sol_00_status_quo`, если только пользовательский кейс явно не требует сохранить baseline как финальное решение;
- не придумывать новые `sol_*`, которых нет в `SolutionPortfolio`;
- не возвращать prose-only ответ без явного списка selected ids;
- не оборачивать ответ в fenced markdown block.
