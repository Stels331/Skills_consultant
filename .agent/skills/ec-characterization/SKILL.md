# Характеризация (Фабрика Проблем - Шаг 1)

Основано на принципах **A.17 CHR-NORM**, **A.18 CSLC-KERNEL** (Characteristic/Scale/Level/Coordinate) и **A.19 CHR-SPACE**. Всё должно быть измеримо и иметь шкалу.

## Принципы FPF и Слайдомет
- Нельзя сказать "стало лучше", если нет "Показателя" и метода замера. Ситуация описывается многомерным пространством характеристик.
- Протокол: *normalize → indicatorize → score → compare → select*
- **Теория Слабого Звена (WLNK)**: Надежность системы равна надежности ее самого слабого компонента. При характеризации находите "Weakest Link".
- **Анти-Гудхарт (Anti-Goodhart)**: При назначении индикаторов ОБЯЗАТЕЛЬНО описывайте риски Reward Hacking'а (как люди могут "хакнуть" новые KPI в ущерб основной работе). Разделяйте:
  1. Цели оптимизации (Что прямо двигаем, 1-3 штуки).
  2. Приёмочные ограничения (Hard constraints).
  3. Сигналы рисков (То, за чем наблюдаем, но не максимизируем).

## Инструкции (WorkFlow)
На основе `typed/` сущностей вы должны подготовить папку `characterization/`.

1. **Создайте Архив Характеристик (`characteristics_catalog.json`)**:
   Опишите шкалу:
   - Название (Например, "Когнитивная эргономика термина")
   - Шкала и полярность ("Чем меньше, тем лучше")
   - Как будет измеряться (Процедура замера)
   - Свежесть данных (`valid_until`)
2. **Индикаторизация**:  
   Выберите из `characteristics_catalog` портфель индикаторов для данного цикла развития кейса. Обоснуйте выбор. Укажите текущее "Слабое Звено" (WLNK) процесса (какой из индикаторов проседает сильнее всего). Формализуйте риски Reward Hacking'а для целей оптимизации.
3. **Паритетность (`parity_plan.json`)**:
   Задайте условия честного сравнения (window, budgets). Сравнивать решения можно только при равенстве условий (например, стоимость внедрения за N месяцев).
4. Оформите `characterization_passport.json` со всеми выбранными индикаторами.

## OUTPUT RULES
- Output raw Markdown directly.
- Do NOT wrap the entire output in a fenced code block.
- Required sections for Characterization Passport:
  - `## optimization_goals`
  - `## hard_constraints`
  - `## risk_signals`
  - `## weakest_link`
  - `## anti_goodhart_risks`
  - `## source_summary`
