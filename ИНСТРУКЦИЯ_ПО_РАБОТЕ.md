# Руководство: Электронный Консультант v3 (FPF Skills)

Этот файл содержит все необходимые инструкции для работы с системой `electronic_consultant_v3` с использованием искусственного интеллекта (Gemini).

---

## 🚀 1. Как запускать проект (Разбор новых кейсов)

Система построена как конвейер (pipeline). Она не висит в памяти постоянно, а запускается **только тогда, когда вам нужно разобрать конкретный документ-кейс от клиента**.

В корне проекта лежит скрипт `run_case.py`. Он умеет читать файлы `.docx`, `.doc`, `.txt`, `.md` и автоматически передавать их на разбор всем агентам.

### Шаг 1: Подготовка терминала
Откройте приложение **Terminal (Терминал)** на вашем Mac и перейдите в папку проекта:
```bash
cd "/Users/stas/Documents/Системное Мышление/Системное мышление/Skills/FPF-skill_2/electronic_consultant_Progect/electronic_consultant_v3"
```

### Шаг 2: Включение интеллекта (Ключ API)
Консультанту нужен "мозг". Подключите Gemini, выполнив следующие команды:
```bash
export GEMINI_API_KEY="ВАШ_КЛЮЧ_GEMINI"
export LLM_MODE="gemini"
```

### Шаг 3: Запуск разбора кейса
Передайте скрипту путь к документу, который нужно разобрать:
```bash
python3 run_case.py "/Users/stas/Downloads/Новый Кейс от Клиента.docx"
```
*(Лайфхак: Напишите `python3 run_case.py ` (с пробелом) и просто перетащите файл мышкой в Терминал — путь вставится сам).*

### Где искать результаты?
Когда скрипт закончит работу (в консоли появится сообщение `🎉 Готово!`), идите в папку `cases/`.
Там появится новая папка с датой (например, `case_20261012_001/reports/`).
Внутри лежат готовые файлы:
* 📄 **Executive_Summary.md** — краткая бизнес-выжимка на 2 страницы для руководства.
* 📄 **Analytical_Full_Report.md** — детальный технический отчет с трассировкой всей логики.

## 🛑 2. Как останавливать проект

Поскольку Электронный Консультант — это не серверный процесс (не веб-сайт), а скрипт-аналитик, **он останавливается сам**, как только дописывает финальный отчет. Вам не нужно его "выключать" или "тушить".

**Экстренная остановка (Прерывание):**
Если скрипт "завис" (например, пропал интернет при обращении к API Gemini) или вы передумали разбирать кейс, просто нажмите в терминале:
`Ctrl + C`
Процесс анализа будет немедленно убит.

## 🧠 3. Как переключать модели и сколько они стоят

Здесь приведена сводка по самым популярным моделям Google Gemini, которые вы можете использовать:

### Сравнение моделей:

| Модель (Название для скрипта) | Для чего идеально подходит | Стоимость за 1 млн токенов (Вход / Выход)* |
| :--- | :--- | :--- |
| **`gemini-2.5-flash`** (по умолчанию) | **"Рабочая лошадка"**. Стабильная, быстрая и умная модель из серии 2.5. Хватает для 90% стандартных кейсов. | Бесплатно (до 15 запросов в мин) / Платно: ~$0.07 / ~$0.30 |
| **`gemini-3.1-pro-preview`** (или `gemini-2.5-pro`) | **"Академик / Senior"**. Новейшая про-модель (3.1). Сильнейшая логика. Видит тонкие противоречия. Думает дольше. Идеальна для сложных кейсов. | Бесплатно (до 2 запросов в мин) / Платно: ~$1.25 / ~$5.00 |
| **`gemini-3.1-flash-lite-preview`** | **"Студент-ассистент"**. Новейшая компактная модель (3.1 Lite). Подходит для простых, массовых задач и грубой фильтрации текстов. | Бесплатно (до 15 запросов в мин) / Платно: ~$0.04 / ~$0.15 |

> **💡 Абсолютно Бесплатное использование (Free Tier):**
> Если вы получили ваш API-ключ в Google AI Studio, то по умолчанию **все эти модели бесплатны** (Free Tier). На бесплатном тарифе есть лишь ограничения по количеству запросов в минуту (как указано в таблице). Для личного разбора кейсов Консультантом этих лимитов более чем достаточно! Деньги (платные тарифы) нужны только если вы делаете высоконагруженный корпоративный продукт.

### Временное переключение (на 1 запуск)
Перед тем как писать `python3 run_case.py ...`, добавьте переменную:
```bash
export GEMINI_MODEL="gemini-3.1-pro-preview"
python3 run_case.py "/путь/к/файлу.docx"
```

### Постоянное переключение (Навсегда)
Если вы хотите, чтобы `gemini-3.1-pro-preview` (или любая другая модель) стала моделью по умолчанию:
1. Откройте файл `app/llm/client.py`.
2. Найдите строку 617 в функции `_call_gemini_responses`:
```python
model = os.environ.get("GEMINI_MODEL", "gemini-2.5-flash")
```
3. Замените название в кавычках:
```python
model = os.environ.get("GEMINI_MODEL", "gemini-3.1-pro-preview")
```
4. Сохраните файл. Теперь скрипт всегда будет обращаться к "тяжелой" логике.

## 🛠 4. Дополнительно: Режим-Заглушка (Локальный тест)

Если вы хотите запустить скрипт "вхолостую" (например, чтобы проверить, что файл `.docx` нормально читается и папки создаются), но **не хотите тратить токены** в Gemini (или у вас нет интернета):

Просто переключите режим на "local" перед запуском:
```bash
export LLM_MODE="local"
python3 run_case.py "/путь/к/файлу.docx"
```
В этом режиме скрипт не пойдет в интернет. Он прогонит папки и создаст отчеты, но вместо умного анализа напишет туда шаблонные фразы-заглушки: `_Generated by local-llm mode._`
